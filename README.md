# AOS Release 8 CLI Helper for Obsidian

Turn your Obsidian notes into a powerful **Alcatel-Lucent Enterprise (ALE) AOS Release 8 CLI editor**.

This plugin provides intelligent autocompletion, syntax highlighting, and a comprehensive command reference for **AOS Release 8.10.R04** (and compatible versions).

![AOS R8 CLI Helper Demo](https://via.placeholder.com/800x400?text=AOS+R8+CLI+Helper+Demo)

## ‚ú® Features

### üöÄ Intelligent Autocompletion
Type CLI commands naturally, and the plugin will suggest the next available keywords, parameters, and options based on the official documentation structure.

- Supports over **2,700 commands** from AOS 8.10.R04.
- Context-aware suggestions (e.g., typing `show ip` suggests `interface`, `route`, `ospf`...).

### üìñ Built-in Command Reference
Need to know what a command does or its exact syntax?
- Press `Ctrl+P` (or `Cmd+P`) and run **"Search AOS R8 Command"**.
- Instantly search the full CLI database.
- View detailed information including:
    - **Description**
    - **Syntax**
    - **Parameters & Descriptions**
    - **Usage Guidelines**
    - **Examples**
    - **Output Definitions** (for show commands)

### üé® Syntax Highlighting
Automatically highlights AOS CLI syntax in code blocks for better readability.

```aosr8
! This is a comment
ip interface "Vlan1" address 192.168.1.1 mask 255.255.255.0 vlan 1
show ip interface
```

To use it, simply use the language tag `aosr8` in your code blocks:

```aosr8
show lanpower ni-priority
```

## üì¶ Installation

### From Community Plugins (Coming Soon)
1. Open Settings > Community Plugins.
2. Search for "AOS R8 CLI Helper".
3. Install and Enable.

### Manual Installation
1. Download the `main.js`, `manifest.json`, and `styles.css` from the latest Release.
2. Create a folder named `aosr8-cli-helper` in your vault's `.obsidian/plugins/` directory.
3. Copy the downloaded files into that folder.
4. Reload Obsidian and enable the plugin.

## üõ†Ô∏è Development

This plugin is built with TypeScript and esbuild.

### Data Source
The command database (`commands_R04.json`) is generated by extracting data directly from the official ALE CLI Reference Guide PDF using custom Python scripts (included in the `extraction_tools` folder of the repo if available, or ask the maintainer).

### Build
1. Clone the repository.
2. Run `npm install`.
3. Run `npm run dev` to start watching for changes, or `npm run build` to build for production.
